Цикл while (продолжение)

Изучим новые трюки, на который способен знакомый цикл while.

Выход из цикла - оператор break

Оператор break осуществляет выход из цикла. Он может использоваться, если в цикле образуются условия, которые несовместимы с его дальнейшим выполнением.

Пример:

number = 0

while number < 5:
    number = number + 1
    if number == 3:    # если number = 3, выходим из цикла
        break
    print(f"number = {number}")

Здесь цикл while проверяет условие number < 5. И пока number не равно 5, предполагается, что значение number будет выводиться на экран. Однако внутри цикла также проверяется другое условие: if number == 3. То есть, если значение number равно 3, то с помощью оператора break выходим из цикла. В итоге на экран будет выведено:

number = 1
number = 2

Бесконечный цикл с break

Используя команду break, можно организовать "бесконечный" цикл (условие которого всегда истинно), и завершать его, подавая команду break в теле цикла при выполнении определённого условия.

Например

while True:
    user_input = input("Для выхода введите слово `bye`: ")
    if user_input == "bye"
        break;
print("Программа завершена.")

Что увидит пользователь:

Для выхода введите слово `bye`: пока
Для выхода введите слово `bye`: adios
Для выхода введите слово `bye`: bye
Программа завершена.

Команда перехода на следующую итерацию - continue

В отличие от оператора break оператор continue выполняет переход к следующей итерации цикла без его завершения.

Например, в предыдущем примере заменим break на continue:

number = 0
while number < 5:
    number += 1
    if number == 3 :    # если number = 3, переходим к новой итерации цикла
        continue
    print(f"number = {number}")

И в этом случае если значение переменной number равно 3, последующие инструкции после оператора continue не будут выполняться в этой итерации:

number = 1
number = 2
number = 4
number = 5

Пустой оператор

Пустой оператор — это оператор, который не выполняет никакого действия. В Пайтоне это оператор pass. Когда компьютер встречает такое, он пропускает пустой оператор и выполняет команды дальше.

Но зачем может понадобиться оператор, который ничего не делает? Его можно вставлять в коде в любое место, где требуется какое-то действие (по синтаксису языка), но при этом мы еще не решили какое действие нужно вставить, или не хотим выполнять в этом месте никакого действия. Поясним это на примере, который был выше в данном уроке.

while True:
    user_input = input("Для выхода введите слово `bye`: ")
    if user_input == "bye"
        break;

Перепишем эту программу, перенеся ввод строки и проверку в условие цикла и избавившись таким образом от необходимости в команде break:

while input("Для выхода введите слово `bye`: ") == "bye":

Казалось бы, в программе есть всё что нужно. В цикле выдается приглашение ввести слово. Введенное слово проверяется на равенство со строкой "bye". Когда введено слово "bye", выполнение цикла заканчивается. Но ЧЕГО-ТО не хватает. По правилам языка, у цикла должно быть тело, а его нет. На строке, следующей за условием цикла, должна быть хотя бы одна команда с отступом слева от первой строки цикла. Исправим программу, поставив в это место пустой оператор:

while input("Для выхода введите слово `bye`: ") == "bye":
    pass
